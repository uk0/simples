CrateDB数据目录发生变动启动故障。

0x00

Crate Version 1.12
Linux Core 4.8
Node Number 20

今天项目要求重新启动crate数据库，因为用户名称发生了变动，所以导致文件没有权限。

1.查看了一番发现时文件的所属发生了变动

#用root用户执行  
chown -R admin:admin /home/tod/data

2.修改完成重新启动。

/crate -d H

3.确认启动成功
